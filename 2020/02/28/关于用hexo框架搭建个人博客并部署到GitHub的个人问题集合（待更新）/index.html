笔者简单记录总结搭建博客中的错误，有些错误忘记记录了，一下为记录的。希望和大家一起交流。

  • 在使用npm install -g hexo-cli来安装hexo框架时，出现如下错误

    简单解读，权限不够无法写入，解决办法就是以管理员身份运行，如下

    sudo npm install -g hexo-cli出现如下便成功，

  • 继续往下，在初始化过程中又遇到了问题（😢），如下：

    原因：hexo初始化必须在一个空目录下，但是并没有，而是直接在家目录下了，所以fatal:~ not empty.....。

    如何解决也很简单，首先通过cd命令cd blog(blog这个文件夹在本地创建的)，然后再进行初始化hexo init。

  • 在进行hexo s时出现Usage: hexo <command>如下图

    解决办法：cd blog再运行hexo s即可。

  • 今早给博客换了个主题，中间又犯了一个小错误。报错WARN No layout: index.html

    解决办法：把主题的文件夹移动到themes文件夹下即可解决。

  • 在给个人博客个性化的时候也遇到了太多的问题，比如刚开始使用tranquilpeak主题，这个主题相对冷门，优化解决上面也有问题，所以遇到了大坑，个人能力有限怎么着也解决不了，只能作罢。于是改换ayer主题，其中没有遇到什么大问题，小问题都谷歌解决，但是中间发现了一个问题，自己的终端代理又出现了问题以及npm也一直报错，怎么着都解决不了，试了很多方法，把node卸载了一遍又一遍（Mac完全卸载node.js）各种谷歌，最后貌似解决了，说说问题和“解决方法”。（也不确定）

    我感觉可能和我的代理有点问题，但是代理的问题也还没解决。以后的文章再给出解决办法吧。先说说这个的解决办法。
