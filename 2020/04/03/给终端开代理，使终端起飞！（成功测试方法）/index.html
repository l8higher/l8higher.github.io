给终端开代理，使终端起飞！（成功测试方法）

我想对于一个命令行使用者来说，在终端用brew或者npm下载时，很多时候会遇到下载超时或者一直处于updating homebrew，那就非常令人恼火。老子下载的包也就十几M,你给老子更新就更新五分钟？？

嘻嘻嘻(#^.^#)



想体验这种速度吗？？？？说实话，速度太快了，还没开始就结束了！

教你哦😯

确定你的shell是bash还是zsh。
一般默认是bash，如果你有换过zsh，那你应该也记得。

Case 1(bash)

  1. 打开终端，输入vi ~/.bashrc，滑到最下方，添加如下
  2. export https_proxy=http://127.0.0.1:7890 http_proxy=http://127.0.0.1:7890 all_proxy=socks5://127.0.0.1:7891

    其中的7890和7891你的可能和我不太一样，可以自己查看自己的代理设置，如果你也使用的clashX，那你可直接复制粘贴。另外，要注意https、http和socks5是不一样的。可查看代理设置。

  3. 依次输入esc :wq保存退出。
  4. 终端输入source ~/.bashrc，使操作生效。

Case 2(zsh)

  1. 打开终端，输入vi ~/.zshrc，滑到最下方，添加如下
  2. export https_proxy=http://127.0.0.1:7890 http_proxy=http://127.0.0.1:7890 all_proxy=socks5://127.0.0.1:7891

    其中的7890和7891你的可能和我不太一样，可以自己查看自己的代理设置，如果你也使用的clashX，那你可直接复制粘贴。另外，要注意https、http和socks5是不一样的。可查看代理设置。

  3. 依次输入esc :wq再按enter。# 保存退出
  4. 终端输入source ~/.zshrc，使操作生效。

测试

网上看到帖子，在终端crul cip.cc来查看ip的方法来验证是否安装成功，是要看情况的，如果你的代理是切换的规则分流模式，那肯定还是你本地ip。那你可以切换到全局直连，再输入命令，来查看。
不过最好的方法就是去自己的GitHub仓库找一个项目下载试试。如果速度起飞，那就OK。
